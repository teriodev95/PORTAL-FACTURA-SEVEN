\section*{Resumen Ejecutivo}

Se entrega el \textbf{Portal de Facturación Electrónica} para Seven Days Gold, un sistema que permite a los socios del gimnasio generar sus facturas CFDI 4.0 de manera autónoma desde cualquier dispositivo.

El sistema conecta directamente con \textbf{EVO} (sistema de gestión del gimnasio) para obtener los datos de las ventas y con \textbf{Facturapi} como Proveedor Autorizado de Certificación (PAC) para el timbrado ante el SAT.

\subsection*{URLs de Acceso}

\begin{tabularx}{\textwidth}{lX}
  \textbf{Portal (Frontend):} & \url{https://seven-days-facturacion.pages.dev} \\[4pt]
  \textbf{API (Backend):} & \url{https://seven-days-api.clvrt.workers.dev} \\
\end{tabularx}

% ============================================================
% ARQUITECTURA
% ============================================================
\section*{Arquitectura del Sistema}

El sistema está construido sobre infraestructura \textbf{Cloudflare}, lo que garantiza disponibilidad global, baja latencia y seguridad.

\begin{tabularx}{\textwidth}{lX}
  \textbf{Frontend} & SvelteKit + Cloudflare Pages \\
  \textbf{Backend API} & Cloudflare Workers + Hono.js \\
  \textbf{Base de Datos} & Cloudflare D1 (SQLite edge) \\
  \textbf{Facturación} & Facturapi.io (PAC) \\
  \textbf{Datos de Ventas} & EVO API (Basic Auth) \\
  \textbf{Validación} & Zod (server-side) \\
\end{tabularx}

\subsection*{Flujo del Sistema}

\begin{enumerate}[leftmargin=1.5cm]
  \item El usuario ingresa su número de ticket en el portal.
  \item El sistema consulta la venta en EVO y muestra los conceptos.
  \item El usuario completa sus datos fiscales (RFC, régimen, C.P., uso CFDI, forma de pago).
  \item El sistema genera el CFDI 4.0 mediante Facturapi y lo timbra ante el SAT.
  \item El usuario descarga su factura en PDF y XML.
\end{enumerate}

% ============================================================
% SEGURIDAD
% ============================================================
\section*{Seguridad Implementada}

\begin{itemize}[leftmargin=1.5cm]
  \item \textbf{API Keys protegidas} --- Almacenadas como Cloudflare Secrets, nunca expuestas en código.
  \item \textbf{Validación server-side} --- Todos los inputs validados con Zod (RFC, C.P., campos requeridos).
  \item \textbf{Sanitización XSS} --- Tags HTML removidos de todos los campos de texto.
  \item \textbf{Protección SQL Injection} --- Drizzle ORM con queries parametrizadas.
  \item \textbf{CORS restringido} --- Solo acepta peticiones del dominio del portal.
  \item \textbf{Secure Headers} --- X-Content-Type-Options, X-Frame-Options, CSP.
  \item \textbf{Factura duplicada} --- Prevención de doble facturación por ticket.
\end{itemize}
